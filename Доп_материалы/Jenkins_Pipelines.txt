Основные определения Pipeline

Пайплайн (Pipeline), конвейер - это определяемая пользователем модель конвейера непрерывной поставки.
Код Пайплайна определяет весь процесс вашей сборки, который обычно включает этапы (stages):
	• сборка приложения					# build
	• тестирование сборки 				# test
	• последующая доставка				# delivery

Кроме того, блок pipeline (конвейера) является ключевой частью синтаксиса декларативного конвейера. 

Node (узел, нода)
	Узлом называется машина, являющаяся частью окружения Jenkins, которая способна выполнять Пайплайн.
	Кроме того, блок узла является ключевой частью синтаксиса Scripted Pipeline.
	
Stage (этап, стадия, ступень)
	Блок этапа определяет концептуально отличающееся подмножество задач, выполняемых на протяжении всего Пайплайна
	(например, этапы «Сборка», «Тестирование» и «Развертывание»), которые используются многими подключаемыми модулями
	для визуализации или представления статуса/прогресса Пайплайна Jenkins. 

Step (шаг)
	Одиночное задание. 	По сути, шаг говорит Дженкинсу, что делать в определенный момент времени (или «шаг» в процессе).
	Например, для выполнения команды оболочки make используйте шаг sh: sh 'make'.
	Когда подключаемый модуль расширяет Pipeline DSL (Domain-specific language), это обычно означает, 
	что подключаемый модуль реализовал новый шаг.
