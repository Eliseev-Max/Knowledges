‼ Можно запустить несколько агентов Bamboo на одной и той же машине.
  Для этого нужно лишь предоставить раздельные домашние директории (каталоги) для установки каждого агента.

Агент может запустить задание (джобу), если его возможности совпадают с требованиями джобы.
Каждое задание (job) наследует требования от отдельных задач (tasks), которые оно содержит.
► Возможности, которые можно определить для агента: ◄
	• исполняемый файл (напр., Maven)
	• JDK (Java Development Kit)
	• клиентское приложение системы контроля версий (напр., Git)
	• пользовательская возможность (custom capability)
	  [ это свойство типа "ключ-значение", определяющее конкретную характеристику агента
	    (например, "operating.system=WindowsXP" или "fast.builds=true")
	  ]

Возможности обычно:
	♦ определяют путь к уже установленному исполняемому файлу,
	♦ должны быть определены в Bamboo,прежде чем Bamboo или его агенты смогут их использовать.

Возможности могут быть определены специально для агента или могут быть общими для всех локальных/удаленных агентов. 
‼ Значение специфичной для агента возможности ПЕРЕОПРЕДЕЛЯЕТ значение общей возможности с тем же именем (если она существует).
